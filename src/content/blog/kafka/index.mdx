## TL;DR

예전부터 카프카를 대용량 데이터 처리할때 쓴다곤 했는데, 당최 어디에 쓰는지 몰랐다.  
이제 알아보고 프로젝트에 적용까지 해보자.

## 카프카란?

- 유튜브 채널(토픽)이 있고, 해당 채널이 영상을 올리면(Pub) 구독한 사람이 정보를 받을 수 있는거다(Sub).
- 메시지가 디스크에도 저장 -> 유실 되지 않습니다.
- 대용량 데이터 처리가 가능하다.

---

### 카프카 내에서의 역할은?

- producer - 데이터를 발행하는, 보내는 역할
- consumer - 데이터를 소비하는 역할
- broker - 위 두 요소 사이에서 전달해주는 역할
- topic - 데이터들 중에서 주제
- partition - topic의 분산 처리를 위한 저장 객체
- zookeeper - 분산 처리를 위한 관리 도구로, topic, partition과 같은 브로커에서 관리하는 메타정보 저장 및 관리
- kafka client - kafka 운영 도구

다음편에서 환경 설정 등 실제 사용을 다뤄보겠다.
