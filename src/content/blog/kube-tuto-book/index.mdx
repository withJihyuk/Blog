## TL;DR

쿠버네티스 교과서 책을 읽고 정리한 내용입니다.

### 쿠버네티스는 파드가 외부 요인으로 인해 종료되는 경우, 새로운 파드를 생성합니다.

- 실제로 모든 파드를 지워도 쿠버네티스는 새로운 파드를 생성합니다.
- 이는 쿠버네티스가 파드의 상태를 관리하고 있기 때문입니다.

파드를 관리 하는 주체는 쿠버니테스의 디플로이먼트 입니다.  
디플로이먼트 컨트롤러는 파드의 상태를 관리하고, 파드의 상태가 변경되면 새로운 파드를 생성합니다.  
이를 역으로 보면, 디플로이먼트 컨트롤러를 제거시, 모든 파드는 terminate 됩니다.

### 파드는 여러 컨테이너로 구성될 수 있습니다.

- 파드는 여러 컨테이너로 구성될 수 있습니다.
- 파드를 한개의 컨테이너로 오해하기 쉽지만, 여러 컨테이너로 구성될 수 있습니다.

### 쿠버네티스에선 서비스로 파드간 통신을 할 수 있다.

- 쿠버네티스에선 서비스로 파드간 통신을 할 수 있습니다.
- 서비스의 selector 를 통해 특정 파드를 선택하고 포트를 지정하여, 해당 포트로 DNS 역할을 하여금 트래픽을 전달 할 수 있습니다.

### 로드밸런서 vs 노드포트

- 노드포트는? 포트에 대해 1:1 매핑을 통해 외부에서 접근할 수 있도록 하는 방법입니다.
- 로드밸런서는? 한개의 포트로 여러 도메인네임을 이용, 여러 서비스에 접근할 수 있도록 하는 방법입니다.

### ExternalName 이란?

- ExternalName 은 외부 도메인 대한 DNS 레코드를 생성하는 방법입니다.
- 외부 도메인에 대한 DNS 레코드를 생성하여, 해당 도메인으로 접근할 수 있도록 합니다.
- 예시: prod DB 주소와 dev DB 주소가 다를 경우, ExternalName 을 통해 그 주소 값만 바꾸어서 배포 상황에서 코드나 환경변수 변경 없이, 배포가 가능해집니다.
- TCP만 사용하는 경우엔 문제가 없지만, HTTP를 사용 하는 경우 헤더가 들어가서... 복잡해집니다.
